Opponible Policy Core

OPC è il **core normativo** dell’ecosistema Hermeticum B.C.E. Qui le regole **non si interpretano**: si applicano.

deterministico UE-first audit-by-design hash-only fail-closed

Cos’è OPC

OPC definisce **policy opponibili** che regolano accesso, evoluzione e limiti operativi dei sistemi AI. Una policy è valida solo se produce sempre lo stesso esito a parità di condizioni.

Cosa NON è

OPC non è linee guida, non è advisory, non è interpretazione umana. Se manca una condizione, il sistema **nega**.

Flusso minimo

Input → Policy → Evidence → Output Ogni decisione lascia una traccia minimale, append-only.

input strutturato verifica ALLOW / DENY audit
